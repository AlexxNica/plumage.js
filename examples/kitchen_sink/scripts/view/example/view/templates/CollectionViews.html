Advanced Usage

ModelViews and CollectionViews can be combined to display complex model hierarchies

In this example a bunch of adhoc view classes are defined to create the view hierarchy. It's a bit messy, but in a real project your views wouldn't be this simple (with more complex rendering, event handlers etc), so you'd define each view class in its own file.

I intend for the definition of the view hierarchy to become more declarative at some point. Rather than creating subViews in initialize, you'll pass some json to a builder to construct the hierarchy.

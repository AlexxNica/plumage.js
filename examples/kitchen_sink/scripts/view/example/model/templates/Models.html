Models inherits from Backbone.Model

Backbone.Model gives us getters, setters, events and async loading. In addition to this, Plumage adds relationships, some more events and unifies the interface for Models and Collections.

Getters and setters

Model values are stored as unstructured JS objects. There's no need to specify field names or types ahead of time.

Example
var model = new Plumage.model.Model({name: 'foo', value: 'bar'});
log('name: ' + model.get('name'));
model.set('name', 'baz');
log('name: ' + model.get('name'));

Events

Models fire events when attributes change, as well as before and after loading.

Example
var model = new Plumage.model.Model();
model.urlRoot = '/my_model';
model.on('all', function(event) {
  log('event: ' + event);
});
model.set({'name': 'foo'});
log('--');
model.load();

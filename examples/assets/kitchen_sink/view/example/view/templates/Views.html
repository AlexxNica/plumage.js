Views inherits from Backbone.View from which we get event delegation and basic DOM rendering.

Plumage adds integration with Handlebars templates, subviews, model binding, and some more hooks (eg onShow/noHide).

Templates

The default rendering logic takes a View's template property and render's it into its DOM element. The template data used in rendering is determined by the getTemplateData method.

Subviews

You can nest views by adding them to a view's subViews property.

Specify the selector property to choose where in the parent view's element the subview will render

Plumage.view.ContainerView

onShow/onHide

onShow and onHide are hooks you can override to execute logic just before a view is shown, or after it's hidden. Just dont' forget to call the super

When using 3rd party views with Plumage, you may find they have issues like rendering problems, or events not triggering. This is because Plumage moves views in and out of the DOM as they're shown and hidden. To fix this, you should override onShow and onHide to add and remove event listeners respectively. If this still doesn't work, you may need to defer the initial rendering of the 3rd party view until onShow is called the first time.

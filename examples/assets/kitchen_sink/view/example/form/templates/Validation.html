There's three places you can perform form validation:

  • In the Plumage View
  • In the Plumage Model
  • On the server

In the Plumage View

In some views you can limit the range of selectable values, eg DateField.

DurationField validates for a number by default.

You can also specify validationRules and validationMessages.

In the Plumage Model

It's possible to do this with Backbone's validation (Override validate and call isValid).

But don't do this. It's too late to provide realtime feedback (like in field validation), and you'll need to validate on the server anyway (protect against manually constructed POST etc.), so the only reason to do this is if you're not saving to a server.

On the Server

Whether or not you validate on the client, you still need to validate on the server.

On save, if the response contains validation errors, the model will trigger 'invalid' instead of 'error'.

{meta: {success: false, validationError: {field: 'message'}}}

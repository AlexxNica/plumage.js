There's three places you can perform form validation:

  • In the Plumage View
  • In the Plumage Model
  • On the server

In the Plumage View

In some views you can prevent selection of invalid values, eg DateField.

You can also specify validationRules and validationMessages.

In the Plumage Model

Override validate and call isValid.

Don't do this. It's too late to provide realtime feedback (like in field validation), and you'll need to validate on the server anyway (protect against manually constructed POST etc.), so the only reason to do this is if you're not saving to a server.

On the Server

Whatever else you do, you must validate on the server.

On save, if the response contains validation errors, the model will trigger 'invalid' instead of 'error'.

{meta: {success: false, validationError: {field: 'message'}}}
